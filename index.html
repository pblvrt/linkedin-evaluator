The LinkedIn audit that tells you what to delete.

A senior-positioning-consultant-grade audit, scored across the 7 things that decide whether profile visitors become clients. Real rewrites, not "be authentic."

01One-time setup

Install the chrome-devtools MCP in Claude Code. Then make sure Chrome is open and you're logged into LinkedIn.

claude mcp add chrome-devtools npx chrome-devtools-mcp@latest

Don't have Claude Code? Install it from claude.com/claude-code.

02Run the audit

Paste this into Claude Code, replacing the URL with your own LinkedIn profile.

Audit my LinkedIn profile using the framework at https://raw.githubusercontent.com/pblvrt/linkedin-evaluator/main/audit.md — my profile is https://linkedin.com/in/YOUR-HANDLE

Claude fetches the framework, drives your already-logged-in Chrome to read the profile, and writes the audit. Takes ~2 minutes.

03What you get

A 0–100 overall score, a verdict, and a weighted scorecard:

Headline

Names the buyer, the outcome, and a credibility marker.

Visuals

Photo + banner. Default LinkedIn blue is throwing money away.

About

Hook in the first two lines, proof, who you help, a real CTA.

Featured

Case study + lead magnet + best post. The closer.

Experience

Outcomes with numbers, not duties from a job description.

Skills

Top 3 align with the headline. Endorsed by the right people.

Content

Cadence, themes, format mix, original substance.

+ Rewrites

3 headlines, a new About opening, banner copy, one post idea.

Then it asks which buyer you actually want to attract and regenerates the rewrites for that buyer specifically.

04Why this works

Most LinkedIn audits tell you to be authentic and tell your story. This one quotes your copy and tells you what to delete. The rubric is built around what makes a service-business profile generate inbound — not what makes HR happy.

How it works under the hood

The audit framework is a single markdown file in the repo. When you paste the magic line, Claude Code fetches it, then uses the chrome-devtools MCP to drive your local Chrome — so it reads your profile from inside your already-logged-in session, never from a server. Your profile content stays in your Claude Code session. Source on GitHub.
